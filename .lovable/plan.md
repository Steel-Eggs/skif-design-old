# Уникальная маркетинговая плашка для каждого товара

## Проблема
Сейчас в `sessionStorage` хранится один общий ключ `marketing-badge-session-v1` с одним `tipIndex` и одним набором чисел. В результате все товары показывают одну и ту же плашку. Нужно: при первом открытии товара случайно выбирать вариант плашки и закреплять его именно за этим товаром на всю сессию.

## React-версия (`src/components/MarketingBadge.tsx`)

1. Использовать переданный `productId` (он уже есть в пропсах и передаётся из `Product.tsx`).
2. Перейти от одного ключа к структуре `{ [productId]: { tipIndex, count, daysAgo, discount } }` под ключом, например, `marketing-badge-by-product-v1`.
3. В `getSessionData(productId)`:
   - прочитать объект из `sessionStorage`;
   - если для этого `productId` записи нет — сгенерировать случайные значения и сохранить;
   - вернуть запись для конкретного товара.
4. В `useMemo` передать `[productId]` в зависимости, чтобы плашка пересчитывалась при переходе между товарами.
5. Если `productId` не передан (например, превью без ID) — поведение не нужно, т.к. плашка теперь только в карточке товара.

## HTML-версия (`public/bitrix/html/product.html`)

1. Получать идентификатор товара из URL: `?id=...` или последний сегмент пути. Если ID не найден — использовать `'default'`.
2. Поменять ключ хранилища на `marketing-badge-by-product-v1` со структурой `{ [productId]: {...} }`.
3. Логика: прочитать объект → если нет записи для текущего ID, сгенерировать `tipIndex/count/daysAgo/discount` случайно и сохранить → отрисовать плашку по этой записи.
4. Старый ключ `marketing-badge-session-v1` можно удалять при первом запуске (опционально), чтобы освободить место.

## Результат
- При открытии товара A — фиксируется одна случайная плашка, она же показывается при повторном заходе в течение сессии.
- При открытии товара B — выбирается своя случайная плашка (может совпасть, может отличаться), и она тоже закрепляется до конца сессии.
- React и HTML работают идентично.
